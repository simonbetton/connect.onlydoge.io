## Repo conventions to preserve

- Package manager/runtime is Bun. Do not introduce npm, pnpm, or yarn metadata.
- TypeScript is strict and ESM. Keep imports using the existing `@/*` alias only where the surrounding file already uses it.
- Formatting is Biome: 2-space indentation, double quotes, no semicolons, line width near 100.
- Architecture direction is documented as `domain <- application <- adapters`. Keep business invariants in domain value objects/entities, workflow logic in application use cases, transport and external IO in adapters, and UI in presentation/routes.
- Tests are Vitest. Prefer table-driven tests for protocol variants and follow existing files such as `src/modules/dogeconnect/adapters/http/elysia-app.test.ts`.
